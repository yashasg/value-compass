# backend

Python services for value-compass. Two long-running processes plus shared
code, all sharing a single virtualenv on the host.

## Architecture

```
Polygon.io
    │ nightly 5PM ET
    ▼
vca-poller (APScheduler)
    │ writes
    ▼
Supabase (Postgres)
    │ reads
    ▼
vca-api (FastAPI) ◄── Cloudflare ◄── iOS App
    │
    └── APNs ──► iOS App
```

## Layout

```
backend/
  api/               # FastAPI application — see api/README.md
  poller/            # APScheduler nightly job — see poller/README.md
  db/                # SQLAlchemy models + Alembic migrations
  common/            # Shared config, constants, utilities
  infra/systemd/     # Host service units for backend processes
  requirements.txt   # Pinned dependencies for both services
```

| Service       | Process               | systemd unit          |
|---------------|-----------------------|-----------------------|
| `backend/api` | `uvicorn api.main:app`| `vca-api.service`     |
| `backend/poller` | `python -m poller`  | `vca-poller.service`  |

## Principles

- `vca-api` reads from Postgres only — it **never** calls Polygon directly.
- `vca-poller` is the only writer to `stock_cache`.
- `backend/common` and `backend/db` are shared between `api` and `poller`
  — they never drift.
- `openapi.json` at the repo root is the contract with the iOS client; it
  is auto-generated by FastAPI and never edited manually.

## OpenAPI contract

FastAPI owns the API contract exposed at `/openapi.json`. After changing API
routes, response models, documented response headers, or schema-version behavior,
regenerate the checked-in contract artifacts:

```bash
PYTHONPATH=backend python3 -m api.export_openapi
```

This writes both `openapi.json` and
`frontend/Sources/Backend/Networking/openapi.json`; backend CI runs the same command with
`--check` so manual edits or stale generated artifacts fail fast.
