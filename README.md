# value-compass

## Structure

```
value-compass/
  backend/
    api/
    poller/
    db/
    common/
    requirements.txt
  ios/
    VCA.xcodeproj
    Sources/
      App/
      Features/
      Networking/
      Models/
  infra/
    systemd/
  .github/
    workflows/
  openapi.json
  README.md
```

## Ownership

| Directory | Purpose |
|---|---|
| `backend/api` | FastAPI application |
| `backend/poller` | APScheduler nightly job |
| `backend/db` | Alembic migrations, shared models |
| `backend/common` | Shared config, constants, utilities |
| `ios/Sources/App` | Entry point, app lifecycle |
| `ios/Sources/Features` | UI screens |
| `ios/Sources/Networking` | SwiftOpenAPIGenerator output |
| `ios/Sources/Models` | Swift data models |
| `infra/systemd` | `vca-api.service`, `vca-poller.service` |
| `.github/workflows` | CI/CD pipelines |
| `openapi.json` | Auto-generated by FastAPI, consumed by SwiftOpenAPIGenerator |

## Principles

- Monorepo — backend, iOS, and infra in one repository.
- `openapi.json` is the contract between backend and iOS, never edited manually.
- `backend/common` and `backend/db` are shared between `api` and `poller` — they never drift.
- Generated Swift networking code lives in `Networking/`, never edited manually.
